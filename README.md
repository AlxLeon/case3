# Greeting App - Веб-приложение Django

Простое веб-приложение на Django, которое позволяет пользователям вводить свои имена и получать персонализированные приветствия.

## Содержание

- [Необходимые условия](#необходимые-условия)
- [Установка](#установка)
- [Запуск приложения](#запуск-приложения)

## Особенности
- Форма ввода имени пользователя
- Персонализированные приветственные сообщения
- Хранение имен пользователей в базе данных
- Обнаружение дубликатов (сообщение "Добро пожаловать снова")
- Защита от CSRF-атак
- Адаптивный веб-дизайн

## Необходимые условия

### Для локальной разработки:
- [pyenv](https://github.com/pyenv/pyenv) - управление версиями Python
- [Poetry](https://python-poetry.org/) - управление зависимостями

### Для работы с Docker:
- Docker Engine
- Docker Compose

## Установка


### Метод 1: Использование Docker (предпочтительно)

1. Соберите образ Docker, если не используете compose:
```bash
docker build -t greeting-app .
```

2. Запустите контейнер:
```bash
docker run -p 8000:8000 greeting-app
```

### Метод 2: Локальная установка с pyenv и Poetry

1. Установите pyenv (если еще не установлен):
```bash
curl https://pyenv.run | bash
```

2. Установите Python 3.12.9:
```bash
pyenv install 3.12.9
```

3. Установите версию Python для директории проекта:
```bash
cd case_3
pyenv local 3.12.9
```

4. Установите poetry-plugin-shell:
```bash
poetry self add poetry-plugin-shell
```

5. Создайте виртуальное окружение и установите зависимости:
```bash
poetry shell
poetry install
```

6. Выполните миграции базы данных:
```bash
python manage.py migrate
```

## Запуск приложения

### Вариант с Docker (предпочтительно):
```bash
# Сборка и запуск с docker-compose (соберёт и запустит)
docker-compose up -d

# Или (без compose) запуск с указанием порта
docker run -p 8000:8000 greeting-app
```

### Локальный вариант:
```bash
# Активируйте виртуальное окружение
poetry shell

# Запустите сервер разработки
python manage.py runserver
```

Приложение будет доступно по адресу `http://127.0.0.1:8000/`


## Переменные окружения

Создайте файл `.env` в корневой директории проекта для локальной разработки:
```bash
SECRET_KEY=ваш_секретный_ключ_здесь
DEBUG=True
```

Добавьте `.env` в `.gitignore` чтобы он не попадал в репозиторий версий.

## Примечания по безопасности

- `SECRET_KEY` никогда не должен быть закоммичен в репозиторий
- Используйте переменные окружения для конфиденциальной конфигурации
- Приложение включает защиту от CSRF по умолчанию
- Миграции базы данных выполняются автоматически
